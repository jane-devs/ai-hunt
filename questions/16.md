# Вопрос 16

Почему ИИ является «усилителем», а не «заменой» разработчика? Что происходит с сильными и слабыми специалистами при появлении ИИ-инструментов?

## Шкала оценивания

| Баллы | Описание |
|-------|----------|
| 0 | Ответ полностью неверный или студент пропустил вопрос. |
| 1–2 | Студент высказывает мнение без аргументации. |
| 3–4 | Понимает, что ИИ не заменяет, но не может объяснить почему. |
| 5–6 | Корректно объясняет: ИИ ускоряет выполнение, но не даёт понимания. Без архитектурного мышления результат хуже. |
| 7–8 | Описывает асимметрию: сильные специалисты становятся ещё сильнее (ускоряются), слабые быстрее упираются в потолок (генерируют больше хаоса). Приводит конкретные примеры: инженер использует ИИ как проверку гипотез; новичок — как источник ответов. |
| 9–10 | Формулирует принцип «ИИ не делает тебя программистом — он делает тебя быстрее тем, кем ты уже являешься». Анализирует рыночные последствия: скорость стала конкурентным преимуществом, отказ от ИИ = добровольное отставание. Связывает с историческими аналогиями (фреймворки, автоматизация). |

## Критерии оценки

- Точность тезиса об «усилителе».
- Описание асимметрии для сильных/слабых специалистов.
- Конкретность аргументов.
- Связь с практикой.
