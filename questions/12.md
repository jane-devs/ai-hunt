# Вопрос 12

Почему важно регулярно делать коммиты при работе с ИИ? Какой подход — по времени или по задачам — более предпочтительный и почему?

## Шкала оценивания

| Баллы | Описание |
|-------|----------|
| 0 | Ответ полностью неверный или студент пропустил вопрос. |
| 1–2 | Знает, что коммиты — это хорошо, но не понимает связи с ИИ. |
| 3–4 | Объясняет важность коммитов в общем смысле, без учёта специфики ИИ-разработки. |
| 5–6 | Понимает, почему при работе с ИИ коммиты критичны: ИИ может неожиданно затронуть большой объём кода, один запрос может «угнать» проект. Коммиты — страховка. |
| 7–8 | Сравнивает два подхода: по времени (раз в час — просто, но коммиты могут быть бессмысленными) и по задачам (после каждой завершённой задачи — изменения осмысленны, легче отследить поломку). Обосновывает предпочтение второго. |
| 9–10 | Формулирует принцип: «коммиты ради коммитов — плохо, фиксировать нужно осмысленные изменения». Описывает рабочий цикл: задача → проверка результата → убеждение в отсутствии лишних изменений → коммит. Связывает с практикой проверки диффа перед применением. |
