# Вопрос 8

Что такое директория `reference/` и каким образом она помогает получать от ИИ более точные результаты?

## Шкала оценивания

| Баллы | Описание |
|-------|----------|
| 0 | Ответ полностью неверный или студент пропустил вопрос. |
| 1–2 | Путает `reference/` с `docs/` или описывает её назначение неверно. |
| 3–4 | Знает, что в `reference/` хранятся внешние материалы, но не понимает механизм влияния на качество генерации. |
| 5–6 | Корректно описывает содержимое: OpenAPI схемы, документация API, куски чужих проектов, исходный код. Понимает базовую идею: давать ИИ реальный контекст вместо угадывания. |
| 7–8 | Объясняет, как использовать `reference/` в запросах: прикладывать файлы, просить анализировать, генерировать код «по аналогии». Приводит конкретный пример: «возьми API из reference и сделай совместимый клиент». |
| 9–10 | Формулирует принцип: `reference/` позволяет ИИ работать с реальными данными, а не «вспоминать». Описывает связку: `reference/` → генерация правил → `docs/` → `.cursorrules`. Объясняет разницу между «ИИ угадывает» и «ИИ работает с источником». |
