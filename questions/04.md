# Вопрос 4

Что такое Claude Code? Чем он принципиально отличается от Cursor по философии и способу использования?

## Шкала оценивания

| Баллы | Описание |
|-------|----------|
| 0 | Ответ полностью неверный или студент пропустил вопрос. |
| 1–2 | Студент путает инструменты или даёт крайне размытое определение. |
| 3–4 | Указывает, что Claude Code работает в терминале, но не может объяснить зачем и в чём преимущество. |
| 5–6 | Корректно описывает: Claude Code — CLI-инструмент для разработки с ИИ через командную строку. Даёт базовое сравнение с Cursor. |
| 7–8 | Полный ответ: объясняет плюсы (гибкость, автоматизация, контроль, встраивание в пайплайны) и минусы (порог входа, нет визуального контекста, требует дисциплины). Понимает, для каких задач какой инструмент лучше. |
| 9–10 | Демонстрирует системное понимание: объясняет разницу философий («ИИ в редакторе» vs «ИИ как часть системы разработки»). Приводит пример пайп-команды. Обосновывает, когда разумно использовать оба инструмента одновременно. |

## Критерии оценки

- Точность определения CLI-инструмента.
- Сравнение с Cursor по существу.
- Понимание сценариев применения.
- Понимание компромиссов.
