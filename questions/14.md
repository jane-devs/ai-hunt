# Вопрос 14

Что такое MCP (Model Context Protocol) и какую задачу он решает при разработке с помощью ИИ?

## Шкала оценивания

| Баллы | Описание |
|-------|----------|
| 0 | Ответ полностью неверный или студент пропустил вопрос. |
| 1–2 | Слышал аббревиатуру, но не может объяснить суть. |
| 3–4 | Знает, что MCP связывает ИИ с чем-то внешним, но не понимает механизм и практическую ценность. |
| 5–6 | Корректно объясняет: MCP — способ подключить к ИИ внешние источники данных и инструменты. MCP-сервер — мост между ИИ и внешним миром (API, файлы, БД, документация). |
| 7–8 | Объясняет разницу «без MCP» и «с MCP»: без MCP — ИИ «вспоминает» API и иногда ошибается; с MCP — обращается к реальному актуальному источнику. Называет 2–3 конкретных примера MCP-серверов (OpenAPI, GitHub, PostgreSQL, docs). |
| 9–10 | Описывает шаги подключения MCP в Cursor: установка сервера, запуск, настройка в интерфейсе (URL, тип сервера). Формулирует принцип: MCP — шаг от «ИИ что-то генерирует» к «ИИ реально работает с системой». Анализирует минусы: не все серверы стабильны, нужно понимать, что подключаешь. |
