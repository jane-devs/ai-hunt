# Вопрос 10

Что такое контекст ИИ? Какие данные входят в контекст при работе в Cursor?

## Шкала оценивания

| Баллы | Описание |
|-------|----------|
| 0 | Ответ полностью неверный или студент пропустил вопрос. |
| 1–2 | Студент думает, что контекст — это просто «история чата». Нет понимания остальных компонентов. |
| 3–4 | Называет 2–3 элемента контекста, но неполно или с путаницей. |
| 5–6 | Корректно перечисляет компоненты: текущий запрос, предыдущие сообщения, открытые файлы, выделенный код, системные инструкции, иногда соседние части проекта. Понимает, что контекст = рабочая память ИИ. |
| 7–8 | Объясняет последствия неправильного контекста: слишком мало — ИИ не понимает зависимости; слишком много — теряет фокус; грязный контекст — хаотичный ответ. Приводит примеры каждого сценария. |
| 9–10 | Формулирует принцип управления контекстом как инженерный навык. Объясняет, что ИИ не «знает проект» — он угадывает поведение системы по тому куску, который ему дали. Формулирует вывод: «ИИ не тупит — он честно работает с тем хаосом, который ты ему дал». |
