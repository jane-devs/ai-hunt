# Вопрос 3

Что такое Cursor? Чем он отличается от обычного Visual Studio Code?

## Шкала оценивания

**Принцип оценки:** накопительный.

| Баллы | Уровень | Описание |
|-------|---------|----------|
| 1–2 | Фактологический | Упоминает, что Cursor — это редактор или IDE, без конкретики. |
| 3–4 | Фактологический | Указывает, что Cursor основан на VS Code и связан с ИИ, но путается в деталях. |
| 5 | Фактологический | Корректно описывает: Cursor — IDE на базе VS Code с встроенным ИИ. Знакомый интерфейс, те же горячие клавиши и расширения. ИИ встроен в процесс, а не как сторонний плагин. |
| 6 | Практическое понимание | Объясняет ключевое отличие: VS Code — редактор, который расширяется плагинами; Cursor — система, изначально спроектированная вокруг ИИ. |
| 7 | Практическое понимание | Перечисляет типичные сценарии использования Cursor: генерация кода, рефакторинг, анализ ошибок, работа с архитектурой. |
| 8 | Практическое понимание | Объясняет, чем Cursor удобнее при работе с большим проектом и несколькими файлами, в отличие от Claude Code в терминале. |
| 9 | Системное понимание | Описывает ограничения Cursor: больше «магии», сложнее автоматизировать, иногда теряется контроль. Понимает, когда использовать Cursor, а когда — другие инструменты. |
| 10 | Системное понимание | Формулирует принцип: «Cursor не делает вас разработчиком — он делает вас быстрее как разработчика». Самостоятельно анализирует компромисс между удобством и контролем. |
