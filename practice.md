## Что нужно реализовать

### Сервис

Ты разрабатываешь три связанных компонента системы:

**Backend API** — REST-сервис с базой данных. Минимальный набор эндпоинтов:

| Метод | Эндпоинт | Описание |
|---|---|---|
| GET | `/employees` | Список сотрудников. Обязательные фильтры: отдел, статус, руководитель. Реализовать пагинацию. |
| GET | `/employees/:id` | Карточка одного сотрудника |
| POST | `/employees` | Добавление нового сотрудника |
| PUT | `/employees/:id` | Изменение данных существующего сотрудника |
| GET | `/departments` | Список отделов с количеством сотрудников |

**Клиентская часть** — страница для просмотра сотрудников с работающими фильтрами по отделу и статусу. Визуальный дизайн не оценивается, функциональность и ui-удобство — да.

**Административная панель** — защищённая страница с формами добавления и редактирования сотрудников. Авторизация может быть простой (логин и пароль в конфиге) или по токену.

### Инфраструктура

- Всё упаковано в **Docker** и запускается через `docker-compose up`
- Задеплоено на сервер и доступно по публичному URL
- Настроен **CI/CD через GitHub Actions или GitLab**: при пуше в `main` проходят тесты (линтер и основной функционал) и происходит автодеплой
- Написаны минимум **3 автотеста** на ключевые эндпоинты

### Стек

Выбираете сами. Обоснование выбора — обязательно (см. требования к файлам ниже).

## Настройка сервера
 
Сервис должен быть доступен по публичному IP или домену. Для этого на сервере настраивается Nginx как обратный прокси перед Docker-контейнером.
 
### Где взять сервер бесплатно
 
Рекомендуемые варианты с бесплатным тарифом:
 
| Платформа | Бесплатный план | Примечание |
|---|---|---|
| Добавить русские или доступные vps | Бесплатные | Проще всего, деплой из GitHub |
 
### Минимальная настройка Nginx
 
Если вы используете VPS (Oracle, Google Cloud или любой другой), нужно настроить Nginx как обратный прокси. Nginx принимает входящие запросы на порту 80 и перенаправляет их в Docker-контейнер.
 
**Шаг 1 — Установка Nginx на сервере:**
 
```bash
sudo apt update
sudo apt install nginx -y
sudo systemctl enable nginx
sudo systemctl start nginx
```
 
**Шаг 2 — Конфигурация виртуального хоста.**
 
Создать файл `/etc/nginx/sites-available/hr-service`:
 
```nginx
server {
    listen 80;
    server_name ВАШ_IP_ИЛИ_ДОМЕН;
 
    # Клиентская часть
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
 
    # API
    location /api/ {
        proxy_pass http://localhost:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
 
    # Административная панель
    location /admin/ {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```
 
> Порты `3000`, `8000`, `3001` — стандартные и примерные. Укажите те которые реально использует ваш `docker-compose.yml`.
 
**Шаг 3 — Активация конфига и перезапуск:**
 
```bash
sudo ln -s /etc/nginx/sites-available/hr-service /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```
 
**Шаг 4 — Проверка:**
 
```bash
curl http://ВАШ_IP/api/employees
```
 
Если получаете ответ от API — всё работает.
 
### Что должно быть открыто в файрволе
 
На большинстве облачных серверов по умолчанию закрыты все порты кроме 22. Нужно открыть:
 
```bash
# UFW (Ubuntu)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```
 
В консоли облачного провайдера также проверьте Security Groups / Firewall Rules — порты 80 и 443 должны быть разрешены для входящего трафика.
 
### Конфигурация которую нужно сдать
 
Файл конфига Nginx добавить в репозиторий по пути `infra/nginx.conf`. Это нужно чтобы проверяющий понял как устроен деплой без доступа к серверу.
 
Также добавить в `docs/README.md` раздел:
 
```markdown
## Сервер
 
- Провайдер: [название]
- Публичный IP / домен: [адрес]
- Nginx проксирует: / → :3000, /api/ → :8000, /admin/ → :3001
- Конфиг: infra/nginx.conf
```

## Частые вопросы

**Можно ли использовать любой ИИ-инструмент?**
Да. Cursor, Claude Code, ChatGPT, GitHub Copilot — любой. Главное документировать промпты.

**Что если ИИ написал весь код, а я только промпты?**
Это нормально и ожидаемо. Оценивается качество управления ИИ, а не количество строк написанных вручную. Но вы должны понимать каждую строку кода и уметь объяснить любое решение.

**Какой стек выбрать?**
Тот с которым вы быстрее всего можете двигаться. Обоснуйте выбор в `DEVELOPMENT.md` — это важнее самого стека.

**Что если деплой не работает?**
Деплой обязателен. Если возникли трудности — опишите в `DEVELOPMENT.md` что пробовали и где застряли. Частичные баллы за попытку лучше нуля за отсутствие.

**Нужен ли красивый дизайн?**
Нет. Оценивается функциональность, не внешний вид.
